Cleanup

Difficulty: Beginner · Estimated time: 15 min

Throughout this training you created networks, security groups, instances, and volumes. This final lesson walks you through renaming every resource to a consistent prefix pattern and then deleting them all so your OVHcloud project is left clean.

1. The Naming Convention

We use the prefix lab- to tag every resource created during the labs. This makes it trivial to list and delete them without touching production resources.

Resource type Original name (example) Renamed to
Network app-network lab-network
Subnet app-subnet lab-subnet
Security group web-server-sg lab-sg-web
Instance web-server-1 lab-instance-web-1
Volume data-vol-1 lab-volume-data-1

2. Rename Resources with the SDK

The OpenStack SDK lets you update the name attribute on most resource types. The helper below renames every resource whose current name matches a known lab name. 

"""cleanup_rename.py — Rename all lab resources to the lab-* pattern."""
from openstack_sdk.auth_manager import ConnectionManager, AuthenticationManager

auth_mgr = AuthenticationManager()
credentials = auth_mgr.load_credentials_from_env()
conn = ConnectionManager(credentials).connect()

PREFIX = "lab-"

# ── Mapping: original name → new name ──────────────────────────
NETWORK_RENAMES = {
    "app-network":          "lab-network",
    "private-network":      "lab-network-private",
    "multi-subnet-network": "lab-network-multi",
}

SUBNET_RENAMES = {
    "app-subnet":      "lab-subnet",
    "private-subnet":  "lab-subnet-private",
    "frontend-subnet": "lab-subnet-frontend",
    "backend-subnet":  "lab-subnet-backend",
}

SG_RENAMES = {
    "web-server-sg":       "lab-sg-web",
    "web-security-group":  "lab-sg-web-group",
    "web-sg":              "lab-sg-web-deploy",
}

INSTANCE_RENAMES = {
    "web-server-1": "lab-instance-web-1",
    "web-server-2": "lab-instance-web-2",
    "web-server-3": "lab-instance-web-3",
}

VOLUME_RENAMES = {
    "data-vol-1":    "lab-volume-data-1",
    "fast-vol-1":    "lab-volume-fast-1",
    "web-data":      "lab-volume-web-data",
    "data-volume-1": "lab-volume-data-v1",
    "data-volume-2": "lab-volume-data-v2",
    "data-volume-3": "lab-volume-data-v3",
}

# ── Rename helpers ─────────────────────────────────────────────
def rename_networks(conn, renames):
    for net in conn.network.networks():
        if net.name in renames:
            print(f"  Network: {net.name} → {renames[net.name]}")
            conn.network.update_network(net.id, name=renames[net.name])

def rename_subnets(conn, renames):
    for sub in conn.network.subnets():
        if sub.name in renames:
            print(f"  Subnet: {sub.name} → {renames[sub.name]}")
            conn.network.update_subnet(sub.id, name=renames[sub.name])

def rename_security_groups(conn, renames):
    for sg in conn.network.security_groups():
        if sg.name in renames:
            print(f"  Security group: {sg.name} → {renames[sg.name]}")
            conn.network.update_security_group(sg.id, name=renames[sg.name])

def rename_instances(conn, renames):
    for srv in conn.compute.servers():
        if srv.name in renames:
            print(f"  Instance: {srv.name} → {renames[srv.name]}")
            conn.compute.update_server(srv.id, name=renames[srv.name])

def rename_volumes(conn, renames):
    for vol in conn.block_storage.volumes():
        if vol.name in renames:
            print(f"  Volume: {vol.name} → {renames[vol.name]}")
            conn.block_storage.update_volume(vol.id, name=renames[vol.name])

print("Renaming lab resources …")
rename_networks(conn, NETWORK_RENAMES)
rename_subnets(conn, SUBNET_RENAMES)
rename_security_groups(conn, SG_RENAMES)
rename_instances(conn, INSTANCE_RENAMES)
rename_volumes(conn, VOLUME_RENAMES)
print("Done — all lab resources now use the lab-* prefix.")

4. Delete All Lab Resources (SDK)

Resources must be deleted in reverse dependency order: volumes first (detach then delete), then instances, security groups, subnets, and finally networks. The script below handles this automatically. 

"""cleanup_delete.py — Delete every lab-* resource in safe order."""
import time

PREFIX = "lab-"

# ── Step 1: Detach and delete volumes ──────────────────────────
print("Step 1/5 — Deleting volumes …")
for vol in conn.block_storage.volumes():
    if not (vol.name and vol.name.startswith(PREFIX)):
        continue
    # Detach if in-use
    if vol.status == "in-use" and hasattr(vol, "attachments"):
        for att in vol.attachments:
            server_id = att.get("server_id")
            if server_id:
                print(f"  Detaching {vol.name} from {server_id}")
                conn.compute.delete_volume_attachment(vol.id, server_id)
        time.sleep(3)  # wait for detachment
    print(f"  Deleting volume: {vol.name}")
    conn.block_storage.delete_volume(vol.id, ignore_missing=True)

# ── Step 2: Delete instances ───────────────────────────────────
print("Step 2/5 — Deleting instances …")
for srv in conn.compute.servers():
    if srv.name and srv.name.startswith(PREFIX):
        print(f"  Deleting instance: {srv.name}")
        conn.compute.delete_server(srv.id, ignore_missing=True)

# Wait for instances to disappear before touching networks
time.sleep(10)

# ── Step 3: Delete security groups ─────────────────────────────
print("Step 3/5 — Deleting security groups …")
for sg in conn.network.security_groups():
    if sg.name and sg.name.startswith(PREFIX):
        print(f"  Deleting security group: {sg.name}")
        conn.network.delete_security_group(sg.id, ignore_missing=True)

# ── Step 4: Delete subnets ─────────────────────────────────────
print("Step 4/5 — Deleting subnets …")
for sub in conn.network.subnets():
    if sub.name and sub.name.startswith(PREFIX):
        print(f"  Deleting subnet: {sub.name}")
        conn.network.delete_subnet(sub.id, ignore_missing=True)

# ── Step 5: Delete networks ───────────────────────────────────
print("Step 5/5 — Deleting networks …")
for net in conn.network.networks():
    if net.name and net.name.startswith(PREFIX):
        print(f"  Deleting network: {net.name}")
        conn.network.delete_network(net.id, ignore_missing=True)

print("\nCleanup complete — all lab-* resources have been removed.")

5. Using the DeploymentEngine Cleanup

If you used the DeploymentEngine during the Deployment lesson, you can pass the resource IDs it returned directly to cleanup_resources(). 

from openstack_sdk.deployment_engine import DeploymentEngine
from openstack_sdk.auth_manager import AuthenticationManager

auth_mgr = AuthenticationManager()
credentials = auth_mgr.load_credentials_from_env()

engine = DeploymentEngine(credentials)

# Use the resource IDs from your deployment result
resource_ids = {
    "volumes":         ["<volume-id-1>", "<volume-id-2>"],
    "instances":       ["<instance-id-1>", "<instance-id-2>"],
    "security_groups": ["<sg-id-1>"],
    "subnets":         ["<subnet-id-1>"],
    "networks":        ["<network-id-1>"],
}

results = engine.cleanup_resources(resource_ids)
for resource, success in results.items():
    status = "deleted" if success else "FAILED"
    print(f"  {resource}: {status}")

6. Terraform Cleanup

If you created resources with Terraform, a single command tears everything down. Terraform reads its state file and deletes resources in the correct order automatically. 

cd terraform
terraform destroy -auto-approve

Verify the state is empty afterwards: 

terraform state list
# Should return nothing

Summary

  • Rename resources to a lab-* prefix so they are easy to identify.
  • Always delete in reverse dependency order: volumes → instances → security groups → subnets → networks.
  • Use DeploymentEngine.cleanup_resources() when you have resource IDs from a previous deployment.
  • For Terraform resources, terraform destroy handles ordering for you.
