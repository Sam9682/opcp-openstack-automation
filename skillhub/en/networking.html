Networking

Difficulty: Intermediate · Estimated time: 20 min

Networks are the foundation of any cloud deployment. In this lesson you will learn how to create private networks, configure subnets with CIDR validation, and set DNS nameservers — using both the Python SDK and Terraform.

1. Network Creation with the SDK

The NetworkManager creates virtual networks and verifies they reach ACTIVE status before returning. 

from openstack_sdk.auth_manager import ConnectionManager
from openstack_sdk.network_manager import NetworkManager
from config.models import NetworkSpec, SubnetSpec, AuthCredentials

credentials = AuthCredentials(
    auth_url="${OS_AUTH_URL}",
    username="${OS_USERNAME}",
    password="${OS_PASSWORD}",
    tenant_name="${OS_TENANT_NAME}",
    region="${OS_REGION_NAME}",
)

conn_manager = ConnectionManager(credentials)
connection = conn_manager.connect()

network_manager = NetworkManager(connection)

# Define network with a subnet
network_spec = NetworkSpec(
    name="app-network",
    admin_state_up=True,
    external=False,
    subnets=[
        SubnetSpec(
            name="app-subnet",
            cidr="192.168.1.0/24",
            ip_version=4,
            enable_dhcp=True,
            dns_nameservers=["8.8.8.8", "8.8.4.4"],
        )
    ],
)

created = network_manager.create_networks([network_spec])
for net in created:
    print(f"Network: {net.name} (ID: {net.id})")

2. Subnet Management

Subnets define the IP address range within a network. Key parameters include CIDR, DHCP, gateway IP, and DNS nameservers. 

# Multiple subnets in one network
network_spec = NetworkSpec(
    name="multi-subnet-network",
    subnets=[
        SubnetSpec(
            name="frontend-subnet",
            cidr="10.0.1.0/24",
            enable_dhcp=True,
            gateway_ip="10.0.1.1",
            dns_nameservers=["8.8.8.8"],
        ),
        SubnetSpec(
            name="backend-subnet",
            cidr="10.0.2.0/24",
            enable_dhcp=True,
            gateway_ip="10.0.2.1",
            dns_nameservers=["8.8.8.8"],
        ),
    ],
)

3. CIDR Validation

The configuration validator checks every CIDR string before deployment. Invalid notation is caught early so you never send a bad request to the API. 

from config.config_manager import ConfigurationManager

manager = ConfigurationManager()
config = manager.load_config("examples/config.yaml")

result = manager.validate_config(config)
if not result.is_valid:
    for err in result.errors:
        print(f"  ✗ {err}")
# Example output:
#   ✗ Invalid CIDR notation: 192.168.1.0/33

4. DNS Nameservers

Subnets can specify custom DNS nameservers. Instances launched in the subnet will receive these via DHCP. 

# In your config.yaml
networks:
  - name: "private-network"
    subnets:
      - name: "private-subnet"
        cidr: "192.168.1.0/24"
        dns_nameservers:
          - "8.8.8.8"
          - "8.8.4.4"
          - "1.1.1.1"

5. Networking with Terraform

The same network can be declared in HCL. Terraform handles dependency ordering automatically. 

# Create a private network
resource "openstack_networking_network_v2" "private_network" {
  name           = var.network_name
  admin_state_up = true
}

# Create a subnet within the network
resource "openstack_networking_subnet_v2" "private_subnet" {
  name            = "${var.network_name}-subnet"
  network_id      = openstack_networking_network_v2.private_network.id
  cidr            = var.subnet_cidr
  ip_version      = 4
  dns_nameservers = var.dns_nameservers
  enable_dhcp     = var.enable_dhcp
}
# variables.tf — network variables
variable "network_name" {
  description = "Name of the private network"
  type        = string
  default     = "private-network"
}

variable "subnet_cidr" {
  description = "CIDR for the subnet"
  type        = string
  default     = "192.168.1.0/24"

  validation {
    condition     = can(cidrhost(var.subnet_cidr, 0))
    error_message = "Must be valid CIDR notation."
  }
}

variable "dns_nameservers" {
  type    = list(string)
  default = ["8.8.8.8", "8.8.4.4"]
}

Summary

  • Use NetworkManager for programmatic network creation with status verification.
  • Define subnets with CIDR, DHCP, gateway, and DNS settings.
  • The validator catches invalid CIDR before any API call.
  • Terraform's HCL provides a declarative alternative with built-in CIDR validation.
