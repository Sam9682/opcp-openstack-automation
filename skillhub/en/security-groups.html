Security Groups

Difficulty: Intermediate · Estimated time: 20 min

Security groups act as virtual firewalls for your instances. Each group contains a set of rules that control inbound (ingress) and outbound (egress) traffic. In this lesson you will create groups, validate rules, and handle both IPv4 and IPv6.

1. Creating a Security Group

The SecurityGroupManager creates groups and their rules in a single call. Names must be unique within a deployment. 

from openstack_sdk.auth_manager import ConnectionManager
from openstack_sdk.security_group_manager import SecurityGroupManager
from config.models import SecurityGroupSpec, SecurityGroupRule, AuthCredentials

credentials = AuthCredentials(
    auth_url="${OS_AUTH_URL}",
    username="${OS_USERNAME}",
    password="${OS_PASSWORD}",
    tenant_name="${OS_TENANT_NAME}",
    region="${OS_REGION_NAME}",
)

conn_manager = ConnectionManager(credentials)
connection = conn_manager.connect()

sg_manager = SecurityGroupManager(connection)

web_sg = SecurityGroupSpec(
    name="web-server-sg",
    description="Allow HTTP, HTTPS, and SSH",
    rules=[
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=22,
            port_range_max=22,
            remote_ip_prefix="0.0.0.0/0",
            ethertype="IPv4",
        ),
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=80,
            port_range_max=80,
            remote_ip_prefix="0.0.0.0/0",
            ethertype="IPv4",
        ),
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=443,
            port_range_max=443,
            remote_ip_prefix="0.0.0.0/0",
            ethertype="IPv4",
        ),
    ],
)

created = sg_manager.create_security_groups([web_sg])
print(f"Created: {created[0].name} (ID: {created[0].id})")

2. Rule Validation

Every rule is validated before it reaches the API. The validator checks:

  • Direction — must be ingress or egress
  • Protocol — must be tcp, udp, icmp, or any
  • Port range — port_range_min <= port_range_max, values 0–65535
  • CIDR — remote_ip_prefix must be valid notation
  • Ethertype — must be IPv4 or IPv6
# The config validator catches rule errors early
from config.config_manager import ConfigurationManager

manager = ConfigurationManager()
config = manager.load_config("examples/config.yaml")
result = manager.validate_config(config)

for err in result.errors:
    print(f"  ✗ {err}")
# Example:
#   ✗ port_range_min must be <= port_range_max in security group web-sg
#   ✗ Invalid CIDR notation in security group rule: 999.0.0.0/8

3. Restricting Access to a Private Network

For database servers you typically restrict access to your private subnet only. 

db_sg = SecurityGroupSpec(
    name="database-sg",
    description="Database — private network only",
    rules=[
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=5432,
            port_range_max=5432,
            remote_ip_prefix="192.168.1.0/24",
            ethertype="IPv4",
        ),
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=3306,
            port_range_max=3306,
            remote_ip_prefix="192.168.1.0/24",
            ethertype="IPv4",
        ),
    ],
)

created = sg_manager.create_security_groups([db_sg])

4. ICMP and IPv6 Rules

ICMP rules (ping) do not use port ranges. IPv6 rules use ::/0 as the wildcard prefix. 

# Allow ping from the 10.0.0.0/8 range
icmp_rule = SecurityGroupRule(
    direction="ingress",
    protocol="icmp",
    remote_ip_prefix="10.0.0.0/8",
    ethertype="IPv4",
)

# Allow HTTPS over IPv6
ipv6_rule = SecurityGroupRule(
    direction="ingress",
    protocol="tcp",
    port_range_min=443,
    port_range_max=443,
    remote_ip_prefix="::/0",
    ethertype="IPv6",
)

5. Querying and Deleting Groups

# Look up by name
sg = sg_manager.get_security_group_by_name("web-server-sg")
if sg:
    print(f"Found: {sg.name} (ID: {sg.id})")

# List all groups
for group in sg_manager.list_security_groups():
    print(f"  - {group.name}")

# Delete when no longer needed
sg_manager.delete_security_group(sg.id)

Summary

  • Security groups are virtual firewalls with ingress/egress rules.
  • Rules are validated for direction, protocol, ports, and CIDR before creation.
  • Restrict database access to private subnets for defense in depth.
  • Use IPv6 rules alongside IPv4 for dual-stack environments.
