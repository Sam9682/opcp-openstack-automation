Compute

Difficulty: Intermediate · Estimated time: 25 min

Compute instances are the workhorses of your infrastructure. This lesson covers instance creation, flavor and image validation, user-data injection, metadata, and status polling — with both the Python SDK and Terraform.

1. Creating Instances with the SDK

The ComputeManager validates flavors and images against the OVHcloud catalog, creates instances, and waits for them to reach ACTIVE status (default timeout: 300 s). 

from openstack_sdk.auth_manager import ConnectionManager
from openstack_sdk.compute_manager import ComputeManager
from config.models import InstanceSpec, AuthCredentials

credentials = AuthCredentials(
    auth_url="${OS_AUTH_URL}",
    username="${OS_USERNAME}",
    password="${OS_PASSWORD}",
    tenant_name="${OS_TENANT_NAME}",
    region="${OS_REGION_NAME}",
)

conn_manager = ConnectionManager(credentials)
connection = conn_manager.connect()

compute = ComputeManager(connection)

specs = [
    InstanceSpec(
        name="web-server-1",
        flavor="s1-4",
        image="Ubuntu 22.04",
        key_name="my-ssh-key",
        network_ids=["private-network"],
        security_groups=["web-security-group"],
        metadata={"role": "web", "env": "production"},
    )
]

instances = compute.create_compute_instances(specs)
for inst in instances:
    print(f"{inst.name}  ID={inst.id}  Status={inst.status}")

2. Flavor and Image Validation

Before sending a create request the manager verifies that the requested flavor and image exist in the OVHcloud catalog. Results are cached to reduce API calls. 

# Common OVHcloud flavors
# s1-2  — 1 vCore,  2 GB RAM
# s1-4  — 1 vCore,  4 GB RAM
# s1-8  — 2 vCores, 8 GB RAM
# b2-7  — 2 vCores, 7 GB RAM
# b2-15 — 4 vCores, 15 GB RAM

# Common images
# Ubuntu 22.04, Ubuntu 20.04

3. User Data Injection

Cloud-init scripts passed via user_data run on first boot. Use them to install packages, start services, or configure the instance. 

user_data_script = """#!/bin/bash
apt-get update
apt-get install -y nginx
systemctl enable nginx
systemctl start nginx
"""

spec = InstanceSpec(
    name="nginx-host",
    flavor="s1-4",
    image="Ubuntu 22.04",
    key_name="my-ssh-key",
    network_ids=["private-network"],
    security_groups=["web-security-group"],
    user_data=user_data_script,
)

instances = compute.create_compute_instances([spec])

4. Metadata

Attach arbitrary key-value metadata to instances for tagging, cost allocation, or automation hooks. 

spec = InstanceSpec(
    name="tagged-instance",
    flavor="s1-2",
    image="Ubuntu 22.04",
    key_name="my-ssh-key",
    network_ids=["private-network"],
    metadata={
        "project": "web-app",
        "environment": "staging",
        "owner": "platform-team",
    },
)

5. Status Polling

After creation the manager polls the API every 5 seconds until the instance reaches ACTIVE or the 300-second timeout expires. If the instance enters ERROR state the fault details are reported immediately. 

# Wait for a specific instance to become ACTIVE
instance = compute.wait_for_instance_active("instance-id", timeout=300)
print(f"Instance is now {instance.status}")

# Query instances
inst = compute.get_instance_by_name("web-server-1")
all_instances = compute.list_instances()

# Delete when done
compute.delete_instance("instance-id")

6. Compute with Terraform

resource "openstack_compute_instance_v2" "instance" {
  count       = var.instance_count
  name        = "${var.network_name}-instance-${count.index + 1}"
  flavor_name = var.instance_flavor
  image_name  = var.instance_image
  key_pair    = var.key_name

  network {
    uuid = openstack_networking_network_v2.private_network.id
  }

  metadata  = var.instance_metadata
  user_data = var.instance_user_data

  depends_on = [
    openstack_networking_subnet_v2.private_subnet
  ]
}
# Apply the Terraform configuration
cd terraform
terraform init
terraform plan -var="instance_count=2"
terraform apply -auto-approve

Summary

  • ComputeManager validates flavors and images before creation.
  • Inject cloud-init scripts via user_data for first-boot automation.
  • Attach metadata for tagging and cost tracking.
  • The SDK polls until ACTIVE (300 s timeout, 5 s interval).
  • Terraform provides a declarative alternative with count-based scaling.
