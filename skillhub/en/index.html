Introduction to OpenStack IaC

Welcome to SkillHub, the hands-on training companion for the opcp-openstack-automation project. This course walks you through Infrastructure as Code on OVHcloud OpenStack — from your first authenticated API call to fully automated multi-resource deployments.

What You Will Learn

By the end of this training path you will be comfortable with three complementary deployment approaches:

  • OpenStack SDK (Python) — programmatic resource management with the project's custom managers
  • Terraform (HCL) — declarative infrastructure definitions with the OpenStack provider
  • Ansible — configuration management and orchestration playbooks

Prerequisites

  • An OVHcloud Public Cloud project with API access
  • Basic familiarity with a terminal / command line
  • Python 3.9+ installed (for SDK lessons)
  • Terraform 1.x installed (for Terraform lessons)

Identify Your Resources — STUDENT_ID

During the labs every resource you create (instances, networks, volumes, security groups…) will be tagged with your student identifier. This makes it easy to find and clean up all your resources once the training is over.

Before you start, export the STUDENT_ID environment variable in your terminal:

export STUDENT_ID=slepetre

Replace slepetre with your own identifier (e.g. your username or trigram). The variable must be set in every terminal session you use for the labs. You can add the line to your ~/.bashrc to make it persistent: 

echo 'export STUDENT_ID=slepetre' >> ~/.bashrc
source ~/.bashrc

All scripts and Terraform configurations in this training reference STUDENT_ID to prefix or tag the resources they create. When the labs are finished, you can filter resources by this tag and delete them in a single pass.

Project Overview

The opcp-openstack-automation repository provides a layered Python SDK that wraps OpenStack APIs into simple, composable managers — authentication, networking, security groups, compute, volumes, and a deployment engine that orchestrates them all. Alongside the SDK you will find Terraform configurations and Ansible playbooks that achieve the same goals declaratively.

Quick Start — Validate Your Setup

Run this small script to confirm your environment is ready:

"""Quick setup validation for opcp-openstack-automation."""
import os
import sys

required_vars = [
    "OS_AUTH_URL",
    "OS_USERNAME",
    "OS_PASSWORD",
    "OS_PROJECT_NAME",
]

missing = [v for v in required_vars if not os.environ.get(v)]

if missing:
    print("Missing environment variables:")
    for var in missing:
        print(f"  - {var}")
    sys.exit(1)

print("All required environment variables are set.")
print(f"Auth URL : {os.environ['OS_AUTH_URL']}")
print(f"Project  : {os.environ['OS_PROJECT_NAME']}")
print("You are ready to begin!")
