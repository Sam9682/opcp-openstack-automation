Authentication

Difficulty: Beginner · Estimated time: 15 min

Before you can manage any OpenStack resource you need a valid, authenticated connection. The openstack_sdk/auth_manager.py module in this project provides several ways to authenticate — from simple environment variables to a full ConnectionManager with automatic token refresh.

1. Environment Variables

The quickest way to get started is to export your OVHcloud credentials as environment variables. The SDK reads them automatically.

Traditional username / password

export OS_AUTH_URL="https://auth.cloud.ovh.net/v3"
export OS_USERNAME="${OS_USERNAME}"
export OS_PASSWORD="${OS_PASSWORD}"
export OS_TENANT_NAME="${OS_TENANT_NAME}"
export OS_REGION_NAME="GRA7"

Application credentials (recommended)

export OS_AUTH_TYPE=v3applicationcredential
export OS_AUTH_URL="${OS_AUTH_URL}"
export OS_IDENTITY_API_VERSION=3
export OS_REGION_NAME="${OS_REGION_NAME}"
export OS_INTERFACE=public
export OS_APPLICATION_CREDENTIAL_ID="${OS_APPLICATION_CREDENTIAL_ID}"
export OS_APPLICATION_CREDENTIAL_SECRET="${OS_APPLICATION_CREDENTIAL_SECRET}"

Application credentials are scoped to a single project and can be revoked independently — they are the preferred approach for automation.

2. Credential Files

For environments where environment variables are inconvenient you can store credentials in a key=value file. Make sure the file permissions are restrictive. 

# credentials.txt — keep permissions tight
chmod 600 credentials.txt
from openstack_sdk.auth_manager import AuthenticationManager

auth_manager = AuthenticationManager()

# Load from a secure file (should have 600 permissions)
credentials = auth_manager.load_credentials_from_file("credentials.txt")

# Authenticate and obtain a connection
connection = auth_manager.authenticate(credentials)
print("Authenticated successfully!")

3. ConnectionManager with Token Refresh

Long-running deployments can outlive a single authentication token (typically 1 hour). The ConnectionManager monitors token expiry and refreshes automatically when fewer than 5 minutes remain. 

from openstack_sdk.auth_manager import ConnectionManager
from config.models import AuthCredentials

credentials = AuthCredentials(
    auth_url="${OS_AUTH_URL}",
    username="${OS_USERNAME}",
    password="${OS_PASSWORD}",
    tenant_name="${OS_TENANT_NAME}",
    region="${OS_REGION_NAME}",
)

conn_manager = ConnectionManager(credentials)

# get_connection() refreshes the token transparently
connection = conn_manager.get_connection()

# Use the connection for long-running operations …
projects = list(connection.identity.projects())
print(f"Found {len(projects)} projects")

# Always close when done
conn_manager.close()

4. Context Manager Usage

Python's with statement guarantees the connection is closed even if an exception occurs. 

from openstack_sdk.auth_manager import ConnectionManager

conn_manager = ConnectionManager(credentials)

with conn_manager as connection:
    servers = list(connection.compute.servers())
    print(f"Running instances: {len(servers)}")
# Connection is automatically closed here

5. Application Credentials

Application credentials let you authenticate without exposing your main password. You create them in the Horizon dashboard or via the CLI. 

# Create an application credential via the OpenStack CLI
openstack application credential create my-automation \
  --description "CI/CD pipeline credential" \
  --unrestricted
from openstack_sdk.auth_manager import AuthenticationManager

auth_manager = AuthenticationManager()

# Environment variables are loaded automatically when
# OS_APPLICATION_CREDENTIAL_ID and OS_APPLICATION_CREDENTIAL_SECRET are set
credentials = auth_manager.load_credentials_from_env()

connection = auth_manager.authenticate(credentials)
print("Authenticated with application credentials!")

6. Error Handling

The module raises AuthenticationError for any failure — missing variables, invalid credentials, or network issues. 

from openstack_sdk.auth_manager import (
    AuthenticationManager,
    AuthenticationError,
)

auth_manager = AuthenticationManager()

try:
    credentials = auth_manager.load_credentials_from_env()
    connection = auth_manager.authenticate(credentials)
except AuthenticationError as exc:
    print(f"Authentication failed: {exc}")
    # Handle gracefully — log, alert, or retry

Summary

  • Use environment variables for quick local development.
  • Use credential files (with chmod 600) when env vars are impractical.
  • Use ConnectionManager for long-running automation — it handles token refresh.
  • Prefer application credentials over username/password for CI/CD.
