Réseaux

Difficulté : Intermédiaire · Durée estimée : 20 min

Les réseaux sont le fondement de tout déploiement cloud. Dans cette leçon, vous apprendrez à créer des réseaux privés, configurer des sous-réseaux avec validation CIDR et définir des serveurs DNS — en utilisant le SDK Python et Terraform.

1. Création de réseau avec le SDK

Le NetworkManager crée des réseaux virtuels et vérifie qu'ils atteignent le statut ACTIVE avant de retourner. 

from openstack_sdk.auth_manager import ConnectionManager
from openstack_sdk.network_manager import NetworkManager
from config.models import NetworkSpec, SubnetSpec, AuthCredentials

credentials = AuthCredentials(
    auth_url="${OS_AUTH_URL}",
    username="${OS_USERNAME}",
    password="${OS_PASSWORD}",
    tenant_name="${OS_TENANT_NAME}",
    region="${OS_REGION_NAME}",
)

conn_manager = ConnectionManager(credentials)
connection = conn_manager.connect()

network_manager = NetworkManager(connection)

# Définir un réseau avec un sous-réseau
network_spec = NetworkSpec(
    name="app-network",
    admin_state_up=True,
    external=False,
    subnets=[
        SubnetSpec(
            name="app-subnet",
            cidr="192.168.1.0/24",
            ip_version=4,
            enable_dhcp=True,
            dns_nameservers=["8.8.8.8", "8.8.4.4"],
        )
    ],
)

created = network_manager.create_networks([network_spec])
for net in created:
    print(f"Réseau : {net.name} (ID : {net.id})")

2. Gestion des sous-réseaux

Les sous-réseaux définissent la plage d'adresses IP au sein d'un réseau. Les paramètres clés incluent le CIDR, le DHCP, l'IP de la passerelle et les serveurs DNS. 

# Plusieurs sous-réseaux dans un même réseau
network_spec = NetworkSpec(
    name="multi-subnet-network",
    subnets=[
        SubnetSpec(
            name="frontend-subnet",
            cidr="10.0.1.0/24",
            enable_dhcp=True,
            gateway_ip="10.0.1.1",
            dns_nameservers=["8.8.8.8"],
        ),
        SubnetSpec(
            name="backend-subnet",
            cidr="10.0.2.0/24",
            enable_dhcp=True,
            gateway_ip="10.0.2.1",
            dns_nameservers=["8.8.8.8"],
        ),
    ],
)

3. Validation CIDR

Le validateur de configuration vérifie chaque chaîne CIDR avant le déploiement. Une notation invalide est détectée tôt pour ne jamais envoyer de requête erronée à l'API. 

from config.config_manager import ConfigurationManager

manager = ConfigurationManager()
config = manager.load_config("examples/config.yaml")

result = manager.validate_config(config)
if not result.is_valid:
    for err in result.errors:
        print(f"  ✗ {err}")
# Exemple de sortie :
#   ✗ Invalid CIDR notation: 192.168.1.0/33

4. Serveurs DNS

Les sous-réseaux peuvent spécifier des serveurs DNS personnalisés. Les instances lancées dans le sous-réseau les recevront via DHCP. 

# Dans votre config.yaml
networks:
  - name: "private-network"
    subnets:
      - name: "private-subnet"
        cidr: "192.168.1.0/24"
        dns_nameservers:
          - "8.8.8.8"
          - "8.8.4.4"
          - "1.1.1.1"

5. Réseaux avec Terraform

Le même réseau peut être déclaré en HCL. Terraform gère automatiquement l'ordre des dépendances. 

# Créer un réseau privé
resource "openstack_networking_network_v2" "private_network" {
  name           = var.network_name
  admin_state_up = true
}

# Créer un sous-réseau dans le réseau
resource "openstack_networking_subnet_v2" "private_subnet" {
  name            = "${var.network_name}-subnet"
  network_id      = openstack_networking_network_v2.private_network.id
  cidr            = var.subnet_cidr
  ip_version      = 4
  dns_nameservers = var.dns_nameservers
  enable_dhcp     = var.enable_dhcp
}
# variables.tf — variables réseau
variable "network_name" {
  description = "Nom du réseau privé"
  type        = string
  default     = "private-network"
}

variable "subnet_cidr" {
  description = "CIDR du sous-réseau"
  type        = string
  default     = "192.168.1.0/24"

  validation {
    condition     = can(cidrhost(var.subnet_cidr, 0))
    error_message = "Doit être une notation CIDR valide."
  }
}

variable "dns_nameservers" {
  type    = list(string)
  default = ["8.8.8.8", "8.8.4.4"]
}

Résumé

  • Utilisez NetworkManager pour la création programmatique de réseaux avec vérification du statut.
  • Définissez les sous-réseaux avec CIDR, DHCP, passerelle et paramètres DNS.
  • Le validateur détecte les CIDR invalides avant tout appel API.
  • Le HCL de Terraform offre une alternative déclarative avec validation CIDR intégrée.
