Volumes

Difficulté : Intermédiaire · Durée estimée : 20 min

Les volumes de stockage bloc fournissent un espace disque persistant qui survit aux redémarrages d'instances et peut être détaché puis rattaché à différentes instances. Cette leçon couvre la création, l'attachement, les volumes bootables et la stratégie de polling avec backoff exponentiel.

1. Création de volumes

Le VolumeManager crée des volumes et attend qu'ils atteignent le statut available avant de retourner. 

from openstack_sdk.auth_manager import ConnectionManager
from openstack_sdk.volume_manager import VolumeManager
from config.models import VolumeSpec, AuthCredentials

credentials = AuthCredentials(
    auth_url="${OS_AUTH_URL}",
    username="${OS_USERNAME}",
    password="${OS_PASSWORD}",
    tenant_name="${OS_TENANT_NAME}",
    region="${OS_REGION_NAME}",
)

conn_manager = ConnectionManager(credentials)
connection = conn_manager.connect()

volume_manager = VolumeManager(connection)

volume_specs = [
    VolumeSpec(name="data-vol-1", size=100, volume_type="classic"),
    VolumeSpec(name="fast-vol-1", size=50, volume_type="high-speed"),
]

volumes = volume_manager.create_and_attach_volumes(volume_specs, {})
for vol in volumes:
    print(f"{vol.name}  Taille={vol.size}Go  Statut={vol.status}")

2. Attachement de volumes aux instances

Passez un champ attach_to référençant un nom d'instance. Le manager vérifie que l'instance est ACTIVE avant l'attachement. 

from openstack_sdk.compute_manager import ComputeManager

compute = ComputeManager(connection)
instances = compute.list_instances()
instances_dict = {inst.name: inst for inst in instances}

volume_specs = [
    VolumeSpec(
        name="web-data",
        size=150,
        volume_type="classic",
        attach_to="web-server-1",
    ),
]

volumes = volume_manager.create_and_attach_volumes(
    volume_specs, instances_dict
)
for vol in volumes:
    print(f"Attaché {vol.name} — Statut : {vol.status}")

3. Volumes bootables

Un volume bootable contient une image OS et peut être utilisé comme disque racine pour une instance. Définissez bootable=True et fournissez un image_id. 

boot_vol = VolumeSpec(
    name="boot-volume",
    size=50,
    volume_type="high-speed",
    bootable=True,
    image_id="image-id-123",  # Remplacez par l'ID d'image réel
)

volumes = volume_manager.create_and_attach_volumes([boot_vol], {})

4. Polling de statut avec backoff exponentiel

Le gestionnaire de volumes utilise un backoff exponentiel lors du polling des changements de statut. L'intervalle commence à 2 secondes et augmente jusqu'à un maximum de 10 secondes. Cela réduit la charge API pendant les périodes chargées. 

# Transitions d'état suivies par le manager :
#   Creating   → Available    (délai : 60 s)
#   Available  → Attaching    (initié par l'appel d'attachement)
#   Attaching  → In-use       (vérification : 30 s)

# Intervalles de polling (backoff exponentiel) :
#   2s → 4s → 8s → 10s (plafonné)

5. Gestion des volumes

# Lister tous les volumes
for vol in volume_manager.list_volumes():
    print(f"  {vol.name}  {vol.size}Go  {vol.status}")

# Détacher un volume
volume_manager.detach_volume("instance-id", "volume-id")

# Supprimer un volume
volume_manager.delete_volume("volume-id")

6. Types de volumes

OVHcloud propose deux types de volumes :

  • classic — stockage bloc standard, adapté aux sauvegardes et archives.
  • high-speed — stockage SSD pour les bases de données et les charges sensibles à la latence.
# Dans votre config.yaml
volumes:
  - name: "data-volume"
    size: 100
    volume_type: "classic"
    attach_to: "web-server-1"

  - name: "db-volume"
    size: 50
    volume_type: "high-speed"
    attach_to: "db-server-1"

Résumé

  • Les volumes fournissent un stockage bloc persistant indépendant du cycle de vie des instances.
  • Le manager vérifie que les instances sont ACTIVE avant l'attachement.
  • Les volumes bootables nécessitent bootable=True et un image_id valide.
  • Le backoff exponentiel (2 s → 10 s) maintient une charge API raisonnable pendant le polling.
