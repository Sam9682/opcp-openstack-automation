Authentification

Difficulté : Débutant · Durée estimée : 15 min

Avant de pouvoir gérer une ressource OpenStack, vous avez besoin d'une connexion valide et authentifiée. Le module openstack_sdk/auth_manager.py de ce projet propose plusieurs méthodes d'authentification — des simples variables d'environnement à un ConnectionManager complet avec rafraîchissement automatique du jeton.

1. Variables d'environnement

La manière la plus rapide de commencer est d'exporter vos identifiants OVHcloud sous forme de variables d'environnement. Le SDK les lit automatiquement.

Nom d'utilisateur / mot de passe traditionnel

export OS_AUTH_URL="https://auth.cloud.ovh.net/v3"
export OS_USERNAME="${OS_USERNAME}"
export OS_PASSWORD="${OS_PASSWORD}"
export OS_TENANT_NAME="${OS_TENANT_NAME}"
export OS_REGION_NAME="GRA7"

Identifiants d'application (recommandé)

export OS_AUTH_TYPE=v3applicationcredential
export OS_AUTH_URL="${OS_AUTH_URL}"
export OS_IDENTITY_API_VERSION=3
export OS_REGION_NAME="${OS_REGION_NAME}"
export OS_INTERFACE=public
export OS_APPLICATION_CREDENTIAL_ID="${OS_APPLICATION_CREDENTIAL_ID}"
export OS_APPLICATION_CREDENTIAL_SECRET="${OS_APPLICATION_CREDENTIAL_SECRET}"

Les identifiants d'application sont limités à un seul projet et peuvent être révoqués indépendamment — c'est l'approche recommandée pour l'automatisation.

2. Fichiers d'identifiants

Pour les environnements où les variables d'environnement ne sont pas pratiques, vous pouvez stocker les identifiants dans un fichier clé=valeur. Assurez-vous que les permissions du fichier sont restrictives. 

# credentials.txt — gardez les permissions strictes
chmod 600 credentials.txt
from openstack_sdk.auth_manager import AuthenticationManager

auth_manager = AuthenticationManager()

# Load from a secure file (should have 600 permissions)
credentials = auth_manager.load_credentials_from_file("credentials.txt")

# Authenticate and obtain a connection
connection = auth_manager.authenticate(credentials)
print("Authenticated successfully!")

3. ConnectionManager avec rafraîchissement du jeton

Les déploiements de longue durée peuvent dépasser la validité d'un jeton d'authentification unique (généralement 1 heure). Le ConnectionManager surveille l'expiration du jeton et le rafraîchit automatiquement lorsqu'il reste moins de 5 minutes. 

from openstack_sdk.auth_manager import ConnectionManager
from config.models import AuthCredentials

credentials = AuthCredentials(
    auth_url="${OS_AUTH_URL}",
    username="${OS_USERNAME}",
    password="${OS_PASSWORD}",
    tenant_name="${OS_TENANT_NAME}",
    region="${OS_REGION_NAME}",
)

conn_manager = ConnectionManager(credentials)

# get_connection() refreshes the token transparently
connection = conn_manager.get_connection()

# Use the connection for long-running operations …
projects = list(connection.identity.projects())
print(f"Found {len(projects)} projects")

# Always close when done
conn_manager.close()

4. Utilisation du gestionnaire de contexte

L'instruction with de Python garantit que la connexion est fermée même si une exception survient. 

from openstack_sdk.auth_manager import ConnectionManager

conn_manager = ConnectionManager(credentials)

with conn_manager as connection:
    servers = list(connection.compute.servers())
    print(f"Running instances: {len(servers)}")
# Connection is automatically closed here

5. Identifiants d'application

Les identifiants d'application vous permettent de vous authentifier sans exposer votre mot de passe principal. Vous les créez dans le tableau de bord Horizon ou via la CLI. 

# Create an application credential via the OpenStack CLI
openstack application credential create my-automation \
  --description "CI/CD pipeline credential" \
  --unrestricted
from openstack_sdk.auth_manager import AuthenticationManager

auth_manager = AuthenticationManager()

# Environment variables are loaded automatically when
# OS_APPLICATION_CREDENTIAL_ID and OS_APPLICATION_CREDENTIAL_SECRET are set
credentials = auth_manager.load_credentials_from_env()

connection = auth_manager.authenticate(credentials)
print("Authenticated with application credentials!")

6. Gestion des erreurs

Le module lève une AuthenticationError en cas d'échec — variables manquantes, identifiants invalides ou problèmes réseau. 

from openstack_sdk.auth_manager import (
    AuthenticationManager,
    AuthenticationError,
)

auth_manager = AuthenticationManager()

try:
    credentials = auth_manager.load_credentials_from_env()
    connection = auth_manager.authenticate(credentials)
except AuthenticationError as exc:
    print(f"Authentication failed: {exc}")
    # Handle gracefully — log, alert, or retry

Résumé

  • Utilisez les variables d'environnement pour le développement local rapide.
  • Utilisez les fichiers d'identifiants (avec chmod 600) lorsque les variables d'environnement ne sont pas pratiques.
  • Utilisez le ConnectionManager pour l'automatisation de longue durée — il gère le rafraîchissement du jeton.
  • Préférez les identifiants d'application au nom d'utilisateur/mot de passe pour le CI/CD.
