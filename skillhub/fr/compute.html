Compute

Difficulté : Intermédiaire · Durée estimée : 25 min

Les instances de calcul sont les piliers de votre infrastructure. Cette leçon couvre la création d'instances, la validation des flavors et images, l'injection de user-data, les métadonnées et le polling de statut — avec le SDK Python et Terraform.

1. Création d'instances avec le SDK

Le ComputeManager valide les flavors et les images par rapport au catalogue OVHcloud, crée les instances et attend qu'elles atteignent le statut ACTIVE (délai par défaut : 300 s). 

from openstack_sdk.auth_manager import ConnectionManager
from openstack_sdk.compute_manager import ComputeManager
from config.models import InstanceSpec, AuthCredentials

credentials = AuthCredentials(
    auth_url="${OS_AUTH_URL}",
    username="${OS_USERNAME}",
    password="${OS_PASSWORD}",
    tenant_name="${OS_TENANT_NAME}",
    region="${OS_REGION_NAME}",
)

conn_manager = ConnectionManager(credentials)
connection = conn_manager.connect()

compute = ComputeManager(connection)

specs = [
    InstanceSpec(
        name="web-server-1",
        flavor="s1-4",
        image="Ubuntu 22.04",
        key_name="my-ssh-key",
        network_ids=["private-network"],
        security_groups=["web-security-group"],
        metadata={"role": "web", "env": "production"},
    )
]

instances = compute.create_compute_instances(specs)
for inst in instances:
    print(f"{inst.name}  ID={inst.id}  Statut={inst.status}")

2. Validation des flavors et images

Avant d'envoyer une requête de création, le manager vérifie que le flavor et l'image demandés existent dans le catalogue OVHcloud. Les résultats sont mis en cache pour réduire les appels API. 

# Flavors OVHcloud courants
# s1-2  — 1 vCore,  2 Go RAM
# s1-4  — 1 vCore,  4 Go RAM
# s1-8  — 2 vCores, 8 Go RAM
# b2-7  — 2 vCores, 7 Go RAM
# b2-15 — 4 vCores, 15 Go RAM

# Images courantes

3. Injection de user-data

Les scripts cloud-init passés via user_data s'exécutent au premier démarrage. Utilisez-les pour installer des paquets, démarrer des services ou configurer l'instance. 

user_data_script = """#!/bin/bash
apt-get update
apt-get install -y nginx
systemctl enable nginx
systemctl start nginx
"""

spec = InstanceSpec(
    name="nginx-host",
    flavor="s1-4",
    image="Ubuntu 22.04",
    key_name="my-ssh-key",
    network_ids=["private-network"],
    security_groups=["web-security-group"],
    user_data=user_data_script,
)

instances = compute.create_compute_instances([spec])

4. Métadonnées

Attachez des métadonnées clé-valeur arbitraires aux instances pour le marquage, l'allocation des coûts ou les hooks d'automatisation. 

spec = InstanceSpec(
    name="tagged-instance",
    flavor="s1-2",
    image="Ubuntu 22.04",
    key_name="my-ssh-key",
    network_ids=["private-network"],
    metadata={
        "project": "web-app",
        "environment": "staging",
        "owner": "platform-team",
    },
)

5. Polling de statut

Après la création, le manager interroge l'API toutes les 5 secondes jusqu'à ce que l'instance atteigne ACTIVE ou que le délai de 300 secondes expire. Si l'instance passe en état ERROR, les détails de l'erreur sont signalés immédiatement. 

# Attendre qu'une instance spécifique devienne ACTIVE
instance = compute.wait_for_instance_active("instance-id", timeout=300)
print(f"L'instance est maintenant {instance.status}")

# Rechercher des instances
inst = compute.get_instance_by_name("web-server-1")
all_instances = compute.list_instances()

# Supprimer quand c'est terminé
compute.delete_instance("instance-id")

6. Compute avec Terraform

resource "openstack_compute_instance_v2" "instance" {
  count       = var.instance_count
  name        = "${var.network_name}-instance-${count.index + 1}"
  flavor_name = var.instance_flavor
  image_name  = var.instance_image
  key_pair    = var.key_name

  network {
    uuid = openstack_networking_network_v2.private_network.id
  }

  metadata  = var.instance_metadata
  user_data = var.instance_user_data

  depends_on = [
    openstack_networking_subnet_v2.private_subnet
  ]
}
# Appliquer la configuration Terraform
cd terraform
terraform init
terraform plan -var="instance_count=2"
terraform apply -auto-approve

Résumé

  • ComputeManager valide les flavors et images avant la création.
  • Injectez des scripts cloud-init via user_data pour l'automatisation au premier démarrage.
  • Attachez des métadonnées pour le marquage et le suivi des coûts.
  • Le SDK interroge jusqu'à ACTIVE (délai de 300 s, intervalle de 5 s).
  • Terraform offre une alternative déclarative avec mise à l'échelle via count.
