Groupes de sécurité

Difficulté : Intermédiaire · Durée estimée : 20 min

Les groupes de sécurité agissent comme des pare-feu virtuels pour vos instances. Chaque groupe contient un ensemble de règles qui contrôlent le trafic entrant (ingress) et sortant (egress). Dans cette leçon, vous créerez des groupes, validerez des règles et gérerez IPv4 et IPv6.

1. Création d'un groupe de sécurité

Le SecurityGroupManager crée les groupes et leurs règles en un seul appel. Les noms doivent être uniques au sein d'un déploiement. 

from openstack_sdk.auth_manager import ConnectionManager
from openstack_sdk.security_group_manager import SecurityGroupManager
from config.models import SecurityGroupSpec, SecurityGroupRule, AuthCredentials

credentials = AuthCredentials(
    auth_url="${OS_AUTH_URL}",
    username="${OS_USERNAME}",
    password="${OS_PASSWORD}",
    tenant_name="${OS_TENANT_NAME}",
    region="${OS_REGION_NAME}",
)

conn_manager = ConnectionManager(credentials)
connection = conn_manager.connect()

sg_manager = SecurityGroupManager(connection)

web_sg = SecurityGroupSpec(
    name="web-server-sg",
    description="Autoriser HTTP, HTTPS et SSH",
    rules=[
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=22,
            port_range_max=22,
            remote_ip_prefix="0.0.0.0/0",
            ethertype="IPv4",
        ),
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=80,
            port_range_max=80,
            remote_ip_prefix="0.0.0.0/0",
            ethertype="IPv4",
        ),
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=443,
            port_range_max=443,
            remote_ip_prefix="0.0.0.0/0",
            ethertype="IPv4",
        ),
    ],
)

created = sg_manager.create_security_groups([web_sg])
print(f"Créé : {created[0].name} (ID : {created[0].id})")

2. Validation des règles

Chaque règle est validée avant d'atteindre l'API. Le validateur vérifie :

  • Direction — doit être ingress ou egress
  • Protocole — doit être tcp, udp, icmp ou any
  • Plage de ports — port_range_min <= port_range_max, valeurs 0–65535
  • CIDR — remote_ip_prefix doit être une notation valide
  • Ethertype — doit être IPv4 ou IPv6
# Le validateur de configuration détecte les erreurs de règles tôt
from config.config_manager import ConfigurationManager

manager = ConfigurationManager()
config = manager.load_config("examples/config.yaml")
result = manager.validate_config(config)

for err in result.errors:
    print(f"  ✗ {err}")
# Exemple :
#   ✗ port_range_min must be <= port_range_max in security group web-sg
#   ✗ Invalid CIDR notation in security group rule: 999.0.0.0/8

3. Restreindre l'accès à un réseau privé

Pour les serveurs de bases de données, on restreint généralement l'accès au sous-réseau privé uniquement. 

db_sg = SecurityGroupSpec(
    name="database-sg",
    description="Base de données — réseau privé uniquement",
    rules=[
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=5432,
            port_range_max=5432,
            remote_ip_prefix="192.168.1.0/24",
            ethertype="IPv4",
        ),
        SecurityGroupRule(
            direction="ingress",
            protocol="tcp",
            port_range_min=3306,
            port_range_max=3306,
            remote_ip_prefix="192.168.1.0/24",
            ethertype="IPv4",
        ),
    ],
)

created = sg_manager.create_security_groups([db_sg])

4. Règles ICMP et IPv6

Les règles ICMP (ping) n'utilisent pas de plages de ports. Les règles IPv6 utilisent ::/0 comme préfixe générique. 

# Autoriser le ping depuis la plage 10.0.0.0/8
icmp_rule = SecurityGroupRule(
    direction="ingress",
    protocol="icmp",
    remote_ip_prefix="10.0.0.0/8",
    ethertype="IPv4",
)

# Autoriser HTTPS en IPv6
ipv6_rule = SecurityGroupRule(
    direction="ingress",
    protocol="tcp",
    port_range_min=443,
    port_range_max=443,
    remote_ip_prefix="::/0",
    ethertype="IPv6",
)

5. Recherche et suppression de groupes

# Rechercher par nom
sg = sg_manager.get_security_group_by_name("web-server-sg")
if sg:
    print(f"Trouvé : {sg.name} (ID : {sg.id})")

# Lister tous les groupes
for group in sg_manager.list_security_groups():
    print(f"  - {group.name}")

# Supprimer quand ce n'est plus nécessaire
sg_manager.delete_security_group(sg.id)

Résumé

  • Les groupes de sécurité sont des pare-feu virtuels avec des règles ingress/egress.
  • Les règles sont validées pour la direction, le protocole, les ports et le CIDR avant la création.
  • Restreignez l'accès aux bases de données aux sous-réseaux privés pour une défense en profondeur.
  • Utilisez des règles IPv6 en complément d'IPv4 pour les environnements double pile.
