Introduction à l'IaC OpenStack

Bienvenue sur SkillHub, le compagnon de formation pratique du projet opcp-openstack-automation. Ce parcours vous guide à travers l'Infrastructure as Code sur OVHcloud OpenStack — de votre premier appel API authentifié jusqu'aux déploiements multi-ressources entièrement automatisés.

Ce que vous apprendrez

À la fin de ce parcours de formation, vous maîtriserez trois approches de déploiement complémentaires :

  • OpenStack SDK (Python) — gestion programmatique des ressources avec les managers personnalisés du projet
  • Terraform (HCL) — définitions d'infrastructure déclaratives avec le fournisseur OpenStack
  • Ansible — gestion de configuration et playbooks d'orchestration

Prérequis

  • Un projet OVHcloud Public Cloud avec accès API
  • Familiarité de base avec un terminal / ligne de commande
  • Python 3.9+ installé (pour les leçons SDK)
  • Terraform 1.x installé (pour les leçons Terraform)

Identifiez vos ressources — STUDENT_ID

Pendant les labs, chaque ressource que vous créez (instances, réseaux, volumes, groupes de sécurité…) sera taguée avec votre identifiant étudiant. Cela permet de retrouver et de nettoyer facilement toutes vos ressources une fois la formation terminée.

Avant de commencer, exportez la variable d'environnement STUDENT_ID dans votre terminal :

export STUDENT_ID=slepetre

Remplacez slepetre par votre propre identifiant (par exemple votre nom d'utilisateur ou trigramme). La variable doit être définie dans chaque session de terminal utilisée pour les labs. Vous pouvez ajouter la ligne à votre ~/.bashrc pour la rendre persistante : 

echo 'export STUDENT_ID=slepetre' >> ~/.bashrc
source ~/.bashrc

Tous les scripts et configurations Terraform de cette formation utilisent STUDENT_ID pour préfixer ou taguer les ressources créées. Une fois les labs terminés, vous pourrez filtrer les ressources par ce tag et les supprimer en une seule passe.

Aperçu du projet

Le dépôt opcp-openstack-automation fournit un SDK Python en couches qui encapsule les API OpenStack dans des managers simples et composables — authentification, réseau, groupes de sécurité, compute, volumes et un moteur de déploiement qui les orchestre tous. En complément du SDK, vous trouverez des configurations Terraform et des playbooks Ansible qui atteignent les mêmes objectifs de manière déclarative.

Démarrage rapide — Validez votre configuration

Exécutez ce petit script pour confirmer que votre environnement est prêt :

"""Quick setup validation for opcp-openstack-automation."""
import os
import sys

required_vars = [
    "OS_AUTH_URL",
    "OS_USERNAME",
    "OS_PASSWORD",
    "OS_PROJECT_NAME",
]

missing = [v for v in required_vars if not os.environ.get(v)]

if missing:
    print("Missing environment variables:")
    for var in missing:
        print(f"  - {var}")
    sys.exit(1)

print("All required environment variables are set.")
print(f"Auth URL : {os.environ['OS_AUTH_URL']}")
print(f"Project  : {os.environ['OS_PROJECT_NAME']}")
print("You are ready to begin!")
