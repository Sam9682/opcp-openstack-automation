SDK & API Cheat Sheet

Difficulté : Intermédiaire · Durée estimée : 10 min

Page de référence rapide pour chaque appel principal du SDK OpenStack utilisé dans les labs. Chaque section correspond à l'un des managers du projet et montre la méthode openstacksdk sous-jacente avec un court exemple.

1. Authentification

Géré par openstack_sdk/auth_manager.py.

ActionAppel SDK
Connexionopenstack.connect(**params)
Valider le tokenconn.authorize()
Fermer la connexionconn.close()
 
import openstack

conn = openstack.connect(
    auth_url="https://auth.cloud.ovh.net/v3",
    project_name="my-project",
    username="user",
    password="secret",
    region_name="GRA7",
    user_domain_name="Default",
    project_domain_name="Default",
)
conn.authorize()   # valide le token
# … utiliser conn …
conn.close()

Application Credentials

ActionAppel SDK
Créer un réseauconn.network.create_network()

# Créer un réseau privé + sous-réseau
network = conn.network.create_network(
    name="my-network",
    admin_state_up=True,
)

subnet = conn.network.create_subnet(
    name="my-subnet",
    network_id=network.id,
    cidr="192.168.1.0/24",
    ip_version=4,
    enable_dhcp=True,
    dns_nameservers=["213.186.33.99"],
    gateway_ip="192.168.1.1",
)

# Lister tous les réseaux
for net in conn.network.networks():
    print(net.name, net.id, net.status)

3. Groupes de sécurité

Géré par openstack_sdk/security_group_manager.py.

ActionAppel SDK
Créer un groupeconn.network.create_security_group()
Créer une règleconn.network.create_security_group_rule()
Lister les groupesconn.network.security_groups()
Supprimer un groupeconn.network.delete_security_group(id)
 
# Créer un groupe de sécurité avec règles SSH + HTTP
sg = conn.network.create_security_group(
    name="web-sg",
    description="Autoriser SSH et HTTP",
)

# Autoriser SSH (port 22)
conn.network.create_security_group_rule(
    security_group_id=sg.id,
    direction="ingress",
    ethertype="IPv4",
    protocol="tcp",
    port_range_min=22,
    port_range_max=22,
    remote_ip_prefix="0.0.0.0/0",
)

# Autoriser HTTP (port 80)
conn.network.create_security_group_rule(
    security_group_id=sg.id,
    direction="ingress",
    ethertype="IPv4",
    protocol="tcp",
    port_range_min=80,
    port_range_max=80,
    remote_ip_prefix="0.0.0.0/0",
)

5. Volumes (Block Storage)

Géré par openstack_sdk/volume_manager.py.

ActionAppel SDK
Créer un volumeconn.block_storage.create_volume()
Obtenir un volumeconn.block_storage.get_volume(id)
Lister les volumesconn.block_storage.volumes()
Supprimer un volumeconn.block_storage.delete_volume(id)
Attacher un volumeconn.compute.create_volume_attachment()
Détacher un volumeconn.compute.delete_volume_attachment()
 
# Créer un volume de 10 Go
volume = conn.block_storage.create_volume(
    name="data-vol",
    size=10,
    volume_type="classic",
)

# Attendre qu'il soit disponible
while conn.block_storage.get_volume(volume.id).status != "available":
    time.sleep(2)

# Attacher à une instance
conn.compute.create_volume_attachment(
    server=server,
    volume_id=volume.id,
)

# Détacher plus tard
conn.compute.delete_volume_attachment(volume.id, server.id)

6. Moteur de déploiement

Géré par openstack_sdk/deployment_engine.py. Orchestre tous les managers dans l'ordre des dépendances.

ActionMéthode
Déploiement completengine.deploy_infrastructure(config)
Rollbackengine.rollback_resources(resources, conn)
Nettoyage manuelengine.cleanup_resources(resource_ids)
 
from openstack_sdk.deployment_engine import DeploymentEngine
from config.config_manager import ConfigManager

config_mgr = ConfigManager()
config = config_mgr.load_config("config.yaml")

engine = DeploymentEngine(credentials)
result = engine.deploy_infrastructure(config)

if result.success:
    print(f"Déployé en {result.duration_seconds:.1f}s")
    print(f"Ressources : {result.created_resources}")
else:
    print(f"Échec : {result.failed_resources}")
    print(f"Orphelins : {result.orphaned_resources}")

7. Nettoyage — Tout supprimer

Les ressources doivent être supprimées dans l'ordre inverse des dépendances : volumes → instances → groupes de sécurité → sous-réseaux → réseaux.

# Suppression rapide des ressources connues
engine.cleanup_resources({
    "volumes":         ["vol-id-1"],
    "instances":       ["srv-id-1"],
    "security_groups": ["sg-id-1"],
    "subnets":         ["sub-id-1"],
    "networks":        ["net-id-1"],
})

Résumé

  • openstack.connect() — point d'entrée pour toute opération.
  • conn.network.* — réseaux, sous-réseaux, groupes de sécurité et règles.
  • conn.compute.* — flavors, images, serveurs, attachements de volumes.
  • conn.block_storage.* — cycle de vie des volumes.
  • DeploymentEngine — orchestre tout ce qui précède avec rollback.
