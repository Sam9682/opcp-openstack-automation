Nettoyage

Difficulté : Débutant · Durée estimée : 15 min

Tout au long de cette formation, vous avez créé des réseaux, des groupes de sécurité, des instances et des volumes. Cette dernière leçon vous guide pour renommer chaque ressource avec un préfixe cohérent, puis les supprimer afin de laisser votre projet OVHcloud propre.

1. La convention de nommage

Nous utilisons le préfixe lab- pour identifier chaque ressource créée pendant les labs. Cela permet de les lister et de les supprimer facilement sans toucher aux ressources de production.

Type de ressource Nom original (exemple) Renommé en
Réseau app-network lab-network
Sous-réseau app-subnet lab-subnet
Groupe de sécurité web-server-sg lab-sg-web
Instance web-server-1 lab-instance-web-1
Volume data-vol-1 lab-volume-data-1

2. Renommer les ressources avec le SDK

Le SDK OpenStack permet de modifier l'attribut name de la plupart des types de ressources. Le script ci-dessous renomme chaque ressource dont le nom correspond à un nom de lab connu. 

"""cleanup_rename.py — Renommer toutes les ressources lab avec le préfixe lab-*."""
from openstack_sdk.auth_manager import ConnectionManager, AuthenticationManager

auth_mgr = AuthenticationManager()
credentials = auth_mgr.load_credentials_from_env()
conn = ConnectionManager(credentials).connect()

PREFIX = "lab-"

# ── Correspondance : nom original → nouveau nom ───────────────
NETWORK_RENAMES = {
    "app-network":          "lab-network",
    "private-network":      "lab-network-private",
    "multi-subnet-network": "lab-network-multi",
}

SUBNET_RENAMES = {
    "app-subnet":      "lab-subnet",
    "private-subnet":  "lab-subnet-private",
    "frontend-subnet": "lab-subnet-frontend",
    "backend-subnet":  "lab-subnet-backend",
}

SG_RENAMES = {
    "web-server-sg":       "lab-sg-web",
    "web-security-group":  "lab-sg-web-group",
    "web-sg":              "lab-sg-web-deploy",
}

INSTANCE_RENAMES = {
    "web-server-1": "lab-instance-web-1",
    "web-server-2": "lab-instance-web-2",
    "web-server-3": "lab-instance-web-3",
}

VOLUME_RENAMES = {
    "data-vol-1":    "lab-volume-data-1",
    "fast-vol-1":    "lab-volume-fast-1",
    "web-data":      "lab-volume-web-data",
    "data-volume-1": "lab-volume-data-v1",
    "data-volume-2": "lab-volume-data-v2",
    "data-volume-3": "lab-volume-data-v3",
}

# ── Fonctions de renommage ─────────────────────────────────────
def rename_networks(conn, renames):
    for net in conn.network.networks():
        if net.name in renames:
            print(f"  Réseau : {net.name} → {renames[net.name]}")
            conn.network.update_network(net.id, name=renames[net.name])

def rename_subnets(conn, renames):
    for sub in conn.network.subnets():
        if sub.name in renames:
            print(f"  Sous-réseau : {sub.name} → {renames[sub.name]}")
            conn.network.update_subnet(sub.id, name=renames[sub.name])

def rename_security_groups(conn, renames):
    for sg in conn.network.security_groups():
        if sg.name in renames:
            print(f"  Groupe de sécurité : {sg.name} → {renames[sg.name]}")
            conn.network.update_security_group(sg.id, name=renames[sg.name])

def rename_instances(conn, renames):
    for srv in conn.compute.servers():
        if srv.name in renames:
            print(f"  Instance : {srv.name} → {renames[srv.name]}")
            conn.compute.update_server(srv.id, name=renames[srv.name])

def rename_volumes(conn, renames):
    for vol in conn.block_storage.volumes():
        if vol.name in renames:
            print(f"  Volume : {vol.name} → {renames[vol.name]}")
            conn.block_storage.update_volume(vol.id, name=renames[vol.name])

print("Renommage des ressources lab …")
rename_networks(conn, NETWORK_RENAMES)
rename_subnets(conn, SUBNET_RENAMES)
rename_security_groups(conn, SG_RENAMES)
rename_instances(conn, INSTANCE_RENAMES)
rename_volumes(conn, VOLUME_RENAMES)
print("Terminé — toutes les ressources lab utilisent le préfixe lab-*.")

3. Lister les ressources par préfixe

Avant de supprimer quoi que ce soit, vérifiez que seules les ressources lab sont ciblées. 

"""cleanup_list.py — Lister chaque ressource dont le nom commence par lab-."""
PREFIX = "lab-"

print("=== Réseaux ===")
for net in conn.network.networks():
    if net.name and net.name.startswith(PREFIX):
        print(f"  {net.name}  (ID : {net.id})")

print("\n=== Sous-réseaux ===")
for sub in conn.network.subnets():
    if sub.name and sub.name.startswith(PREFIX):
        print(f"  {sub.name}  (ID : {sub.id})")

print("\n=== Groupes de sécurité ===")
for sg in conn.network.security_groups():
    if sg.name and sg.name.startswith(PREFIX):
        print(f"  {sg.name}  (ID : {sg.id})")

print("\n=== Instances ===")
for srv in conn.compute.servers():
    if srv.name and srv.name.startswith(PREFIX):
        print(f"  {srv.name}  (ID : {srv.id})  Statut : {srv.status}")

print("\n=== Volumes ===")
for vol in conn.block_storage.volumes():
    if vol.name and vol.name.startswith(PREFIX):
        print(f"  {vol.name}  (ID : {vol.id})  Statut : {vol.status}")

4. Supprimer toutes les ressources lab (SDK)

Les ressources doivent être supprimées dans l'ordre inverse des dépendances : d'abord les volumes (détacher puis supprimer), puis les instances, les groupes de sécurité, les sous-réseaux et enfin les réseaux. 

"""cleanup_delete.py — Supprimer toutes les ressources lab-* dans l'ordre sûr."""
import time

PREFIX = "lab-"

# ── Étape 1 : Détacher et supprimer les volumes ───────────────
print("Étape 1/5 — Suppression des volumes …")
for vol in conn.block_storage.volumes():
    if not (vol.name and vol.name.startswith(PREFIX)):
        continue
    # Détacher si en cours d'utilisation
    if vol.status == "in-use" and hasattr(vol, "attachments"):
        for att in vol.attachments:
            server_id = att.get("server_id")
            if server_id:
                print(f"  Détachement de {vol.name} du serveur {server_id}")
                conn.compute.delete_volume_attachment(vol.id, server_id)
        time.sleep(3)  # attendre le détachement
    print(f"  Suppression du volume : {vol.name}")
    conn.block_storage.delete_volume(vol.id, ignore_missing=True)

# ── Étape 2 : Supprimer les instances ─────────────────────────
print("Étape 2/5 — Suppression des instances …")
for srv in conn.compute.servers():
    if srv.name and srv.name.startswith(PREFIX):
        print(f"  Suppression de l'instance : {srv.name}")
        conn.compute.delete_server(srv.id, ignore_missing=True)

# Attendre que les instances disparaissent avant de toucher aux réseaux
time.sleep(10)

# ── Étape 3 : Supprimer les groupes de sécurité ───────────────
print("Étape 3/5 — Suppression des groupes de sécurité …")
for sg in conn.network.security_groups():
    if sg.name and sg.name.startswith(PREFIX):
        print(f"  Suppression du groupe de sécurité : {sg.name}")
        conn.network.delete_security_group(sg.id, ignore_missing=True)

# ── Étape 4 : Supprimer les sous-réseaux ──────────────────────
print("Étape 4/5 — Suppression des sous-réseaux …")
for sub in conn.network.subnets():
    if sub.name and sub.name.startswith(PREFIX):
        print(f"  Suppression du sous-réseau : {sub.name}")
        conn.network.delete_subnet(sub.id, ignore_missing=True)

# ── Étape 5 : Supprimer les réseaux ───────────────────────────
print("Étape 5/5 — Suppression des réseaux …")
for net in conn.network.networks():
    if net.name and net.name.startswith(PREFIX):
        print(f"  Suppression du réseau : {net.name}")
        conn.network.delete_network(net.id, ignore_missing=True)

print("\nNettoyage terminé — toutes les ressources lab-* ont été supprimées.")

5. Utiliser le nettoyage du DeploymentEngine

Si vous avez utilisé le DeploymentEngine pendant la leçon Déploiement, vous pouvez passer les IDs de ressources retournés directement à cleanup_resources(). 

from openstack_sdk.deployment_engine import DeploymentEngine
from openstack_sdk.auth_manager import AuthenticationManager

auth_mgr = AuthenticationManager()
credentials = auth_mgr.load_credentials_from_env()

engine = DeploymentEngine(credentials)

# Utilisez les IDs de ressources de votre résultat de déploiement
resource_ids = {
    "volumes":         ["<volume-id-1>", "<volume-id-2>"],
    "instances":       ["<instance-id-1>", "<instance-id-2>"],
    "security_groups": ["<sg-id-1>"],
    "subnets":         ["<subnet-id-1>"],
    "networks":        ["<network-id-1>"],
}

results = engine.cleanup_resources(resource_ids)
for resource, success in results.items():
    status = "supprimé" if success else "ÉCHOUÉ"
    print(f"  {resource} : {status}")

6. Nettoyage Terraform

Si vous avez créé des ressources avec Terraform, une seule commande supprime tout. Terraform lit son fichier d'état et supprime les ressources dans le bon ordre automatiquement. 

cd terraform
terraform destroy -auto-approve

Vérifiez que l'état est vide ensuite : 

terraform state list
# Ne devrait rien retourner

Résumé

  • Renommez les ressources avec un préfixe lab-* pour les identifier facilement.
  • Supprimez toujours dans l'ordre inverse des dépendances : volumes → instances → groupes de sécurité → sous-réseaux → réseaux.
  • Utilisez DeploymentEngine.cleanup_resources() quand vous avez les IDs de ressources d'un déploiement précédent.
  • Pour les ressources Terraform, terraform destroy gère l'ordre pour vous.
